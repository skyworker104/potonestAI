# PhotoNest AI — 내 컴퓨터에서 돌아가는 AI 사진 관리

구글포토처럼 사진을 관리하면서, 사진은 단 한 장도 외부로 보내지 않는
**로컬 사진 관리 + AI 자연어 검색** 웹 애플리케이션입니다.
Mac / Windows / Linux / 라즈베리파이에서 동작하며, 같은 와이파이의
태블릿·폰 브라우저에서도 접속할 수 있습니다.

## 주요 기능

| 기능 | 설명 |
|------|------|
| 🔍 AI 자연어 검색 | "바닷가 사진 찾아줘", "작년 야경" — 한국어 문장으로 검색 (SigLIP2 다국어, 번역 없이 이해) |
| 🔤 사진 속 글자 검색 | 스크린샷·문서·간판 등에 적힌 텍스트로 검색 (OCR, 선택) |
| 🎙️ 음성 대화 | 검색·앨범·슬라이드쇼까지 음성으로 조작 (Chrome) |
| 🖼️ 타임라인 | 월별 그룹, 수만 장도 가볍게 (지연 로딩) |
| 📁 앨범 | 만들기/이름변경/삭제, 사진 추가/제거, 멀티선택 |
| ⭐ 즐겨찾기 | 한 번에 모아보기 |
| ▶ 슬라이드쇼 | 전체화면 자동재생 (Ken Burns 효과) |
| 👤 인물 | 얼굴 자동 인식·동일 인물 그룹핑, 이름 붙이면 "철수 사진 보여줘"로 검색 |
| 🗺️ 지도 | 촬영 위치 클러스터 지도, 사진별 상세 위치 (ℹ️ 토글) |
| 🗑️ 휴지통 | 삭제/복원/영구삭제 — 실수 방지 |
| 🧹 중복 정리 | 내용 해시 기반 동일 사진 그룹핑, 한 번에 정리 |
| 📦 구글 테이크아웃 | Takeout 폴더를 넣으면 사이드카 JSON의 촬영시각·GPS 자동 인식 |
| 📱 폰 업로드 | QR로 접속하는 모바일 업로드 페이지 — 원본 무손실 전송, 중복 자동 스킵, HEIC 지원 |
| 🔄 자동 백업 | 내장 WebDAV 서버 — PhotoSync/FolderSync 앱 연결 시 "와이파이 진입하면 자동 업로드" |

## 설치 (한 번만)

```bash
# macOS / Linux / 라즈베리파이 — 기기 사양을 보고 AI 방식을 자동 제안
./install.sh          # 대화형: 전체(torch)/경량(ONNX)/없음 중 선택
./install.sh --full   # 전체 — torch + SigLIP2 다국어 (PC/고성능, ~2GB)
./install.sh --lite   # 경량 — ONNX CLIP (저사양/ARM/태블릿, ~200MB)
./install.sh --no-ai  # AI 없이 메타데이터/OCR/얼굴 검색만

# Windows: install.bat 더블클릭
# 안드로이드 태블릿(Termux): 아래 "안드로이드 태블릿에서 서버로" 참고
```

Python 3.9 이상만 있으면 됩니다. 설치가 끝나면 `run.sh`(Windows는 `run.bat`)가 생성됩니다.

두 AI 방식의 차이:

| | 전체 (torch) | 경량 (ONNX) |
|---|---|---|
| 이미지 의미검색 모델 | SigLIP2 (다국어 네이티브) | CLIP ViT-B/32 (영어) |
| 한국어 질의 | 직접 이해 | OpenRouter/내장 사전이 자동 번역 |
| 다운로드 | ~2GB | ~200MB (패키지+모델) |
| 최소 RAM | 8GB 권장 | 2GB에서도 동작 |
| 코멘트 문장검색 | ✅ | ❌ (문장 유사도 모델이 torch 기반) |

임베딩은 백엔드별로 비호환이라 방식을 바꾸면 자동으로 전량 재색인됩니다
(백그라운드에서 천천히 — 아래 "메모리 인식 색인" 참고).

## 사용

```bash
./run.sh
```

1. 사진/동영상을 `photos/` 폴더에 넣으면 자동 색인됩니다 (하위 폴더 포함, 구글 테이크아웃 폴더 그대로 OK)
2. 브라우저(Chrome 권장)에서 <http://localhost:8765> 접속
3. 태블릿/다른 기기에서는 `http://<컴퓨터IP>:8765`

옵션 (run.sh에서 수정):
- `PHOTOS_DIR=/path/to/photos` — 다른 사진 폴더 사용
- `AI_SEARCH=0` — AI 검색 끄기
- `ANTHROPIC_API_KEY=...` — Claude로 질의 해석

## 대화형 검색을 위한 로컬 LLM (선택, 권장)

자연어 발화를 정교하게 분석해 검색 조건(내용·날짜·인물·미디어 종류)을 만들고,
"그 중에 작년 것만" 같은 **후속 질문(대화 맥락)**까지 이해하려면 로컬 LLM을 연결하세요.
설치·키 없이 **내 컴퓨터에서만** 동작하며, 없으면 자동으로 휴리스틱/Claude로 폴백합니다.

지원: OpenAI 호환 API를 제공하는 모든 로컬 LLM — **LM Studio**, **Ollama**, llama.cpp 등.

- **LM Studio**: 앱에서 모델을 로드하고 Developer → **Start Server**(기본 포트 1234).
  서버가 켜져 있으면 PhotoNest가 자동 인식합니다.
  - 권장: 모델을 **계속 로드된 상태로 유지**하고 GPU 풀 오프로드로 로드하세요
    (`lms load <모델> --gpu max --ttl 7200`). 그러지 않으면 호출마다 콜드 로딩으로 느려집니다.
  - **모델 선택이 중요합니다**: 빠른 응답을 원하면 **비(非)-reasoning instruct 모델**을 쓰세요
    (예: `qwen2.5-3b-instruct`, `gemma-2-2b-it`, `llama-3.2-3b-instruct` — 응답 1~3초).
    Gemma-3n/생각형(reasoning) 모델은 의도 분석 정확도는 높지만 "생각" 과정 때문에
    한 번에 20~100초까지 걸릴 수 있습니다(어댑터가 동작은 하나 느림).
- **Ollama**: `ollama serve` 후 `ollama run <모델>` (포트 11434 자동 인식).
- 직접 지정: `LOCAL_LLM_BASE=http://localhost:1234/v1 LOCAL_LLM_MODEL=<모델id>`

엔진 동작 상태는 화면 상단(색인 상태 옆)에서 확인할 수 있습니다.

## OpenRouter로 클라우드 모델 쓰기 (선택)

로컬 LLM을 띄우기 어렵거나(저사양 기기·GPU 없음) 더 정확한 해석을 원하면,
[OpenRouter](https://openrouter.ai) 키 하나로 Claude·Gemini·GPT·Llama 등
수백 개 클라우드 모델을 대화형 검색 엔진으로 쓸 수 있습니다.
발화 해석(의도·인물·미디어 종류)에만 쓰이며, **사진 자체는 절대 외부로 전송되지 않습니다**
(이미지 검색은 항상 로컬 CLIP이 담당).

### 앱에서 설정 (권장)

1. <https://openrouter.ai/keys> 에서 API 키 발급 (`sk-or-v1-...`)
2. 채팅 패널 상단 **⚙️ (AI 엔진 설정)** 버튼 클릭
3. 엔진에서 **OpenRouter**(또는 **자동**) 선택 → **API 키** 붙여넣기 → **모델** 선택
   - 추천 모델(가볍고 빠르고 저렴): `google/gemini-2.5-flash-lite`,
     `anthropic/claude-haiku-4.5`, `openai/gpt-4o-mini` — 목록에서 고르거나 직접 입력
4. **연결 테스트**로 키·모델 확인 후 **저장**

설정은 `data/app/settings.json`에 저장되며, 키는 화면에 마스킹 표시됩니다.
저장 후 상단 상태줄에 `🌐 <모델명>`으로 사용 중인 엔진이 표시됩니다.

### 엔진 우선순위

⚙️ 설정의 **엔진 선택**에 따라 동작합니다(어떤 엔진이 실패해도 항상 휴리스틱으로 폴백):

| 모드 | 동작 |
|------|------|
| **자동** | OpenRouter(키 있으면) → 로컬 LLM → Claude → 휴리스틱 |
| **OpenRouter** | OpenRouter만 사용 |
| **로컬 LLM** | LM Studio/Ollama 자동탐지만 사용 |
| **Claude** | `ANTHROPIC_API_KEY`만 사용 |

> 🔒 **보안**: API 키는 `data/app/settings.json`에 평문 저장되므로 이 파일은
> `.gitignore`로 제외되어 있습니다. 저장소를 공유할 때 키가 유출되지 않습니다.

### 지역명 검색 + 피드백 학습

- **지역명은 위치(GPS)로 검색**: "제주도 사진"처럼 지역명을 말하면, 이미지 의미 검색
  (제주 해변과 다른 해변을 구분 못 함) 대신 **사진의 GPS 좌표로 그 지역 안인지 판정**해
  정확히 찾습니다. 지원 지역: 제주·부산·서울·강릉·경주·여수·통영·괌 등(확장 가능).
- **결과 교정 피드백**: 검색 결과에 다른 지역이 섞였을 때 채팅에 "제주도 것만 보여줘",
  "위치로 다시 찾아줘"라고 하면 **직전 검색을 위치 기준으로 바로잡고**, 그 교정을
  스킬로 학습해 다음부터 반영합니다.
- **동네 단위 지명 검색 (역지오코딩)**: 사진 GPS를 오프라인
  [GeoNames](https://www.geonames.org) 데이터(CC-BY 4.0, 최초 1회 ~12MB 다운로드)로
  가장 가까운 지명에 매핑해 저장합니다 — "수서동 사진", "하갓냐에서 찍은 사진"처럼
  사전에 없는 동네·해외 도시 이름으로도 검색됩니다. 라이트박스 위치 표시에도 지명이
  함께 나옵니다. (일부 리(里) 단위는 GeoNames에 한글 이름이 없어 시·도명으로 보완)

### 이름 연관검색 (앨범·코멘트·폴더명)

"씨메르 사진"처럼 **이미지 모델이 알 수 없는 고유명사**는 사용자가 붙인 이름에서
찾습니다 — 앨범명(가장 높은 가중치)·코멘트·자동 캡션·지명·폴더/파일명을 단어
일치로 검사해 이미지 검색 결과 위에 랭크합니다. 음성인식 변형("씨메르"↔"시메르")과
오타도 한 글자까지 허용합니다. 앨범명이 일치하면 관련도 컷(상위 60장)의 예외로
**그 앨범 전체**가 반환됩니다.

### 검색 스킬 (쓸수록 빨라짐)

로컬 LLM이 한 번 해석한 검색은 **스킬로 자동 저장**됩니다. 이후 비슷한 질문이 오면
LLM을 다시 부르지 않고 그 스킬을 **즉시 재사용**합니다(특히 reasoning 모델의 느린 응답을 우회).

- **의미 기반 매칭**: "바닷가 사진"을 한 번 배우면 "해변에서 찍은 거", "바다 풍경"도 즉시 처리.
  핵심 주제어만 비교하므로 "~사진 보여줘" 같은 문장 구조에 속지 않습니다.
- **변동 요소는 매번 재계산**: 날짜("3년 전")·인물·미디어 종류는 캐싱하지 않아 시간이 지나도 안전.
- **보수적 재사용**: 확실히 같은 의미일 때만 스킬을 쓰고, 애매하면 LLM이 정확히 처리합니다.
- 채팅 패널의 **❔ 도움말** 버튼(또는 "도움말"이라고 말하기)에서 예시 질문과 배운 스킬을
  칩으로 보고, 눌러서 바로 검색하거나 ✕로 삭제할 수 있습니다.
- AI 응답에는 처리 방식 배지(⚡ 스킬 / 🧠 로컬 LLM)가 표시됩니다.

### 사진 속 글자 검색 (OCR, 선택)

스크린샷·문서·간판 등에 적힌 텍스트를 자동 인식해 검색 대상에 포함합니다.
`tesseract`(brew install tesseract tesseract-lang / apt install tesseract-ocr tesseract-ocr-kor)가
있으면 자동 활성화되고, 없으면 조용히 비활성화됩니다(설치 필수 아님).

### 신규 사진 자동 캡션 (선택, 비전 LLM 필요)

"웃고 있는 사람", "생일 파티"처럼 상황·관계를 묘사하는 질의를 더 잘 찾도록,
LM Studio에 **비전(이미지 이해) 모델**을 올리고 `VISION_LLM_MODEL`을 지정하면
새로 추가되는 사진마다 자동으로 한국어 캡션을 생성해 검색에 반영합니다.

```bash
export VISION_LLM_MODEL=qwen/qwen2.5-vl-7b   # LM Studio에 로드된 비전 모델 id
```

- **기존 라이브러리는 소급 적용되지 않습니다** — 비전 모델은 사진 1장에 수십 초가 걸려
  라이브러리 전체를 한 번에 처리하면 매우 오래 걸립니다(수만 장이면 며칠 단위).
  이후 새로 추가되는 사진에만 자동 적용됩니다.
- 설정하지 않으면 완전히 비활성화되어 색인 속도에 영향이 없습니다.

## 안드로이드 태블릿에서 서버로 (Termux)

RAM 4GB급 태블릿(Galaxy Tab A8 등)을 PhotoNest 단독 서버로 쓸 수 있습니다.
Termux 안에 proot Ubuntu(루팅 불필요)를 깔고 경량(ONNX) AI로 동작합니다.

```bash
# 1) F-Droid에서 Termux 설치 (Play스토어판은 구버전)
# 2) Termux에서:
pkg install -y git
git clone <저장소URL> photonest && cd photonest
bash scripts/install-termux.sh
# 3) 시작:
~/run-photonest.sh          # tmux 세션 'photonest'에서 서버 실행
tmux attach -t photonest    # 로그 화면 보기 (Ctrl+b, d로 빠져나옴)
```

- 사진 위치: 내부 공유저장소는 `/media/shared`, SD카드는 `/media/sdcard`로
  proot 안에 마운트됩니다 — `PHOTOS_DIR=/media/sdcard/DCIM` 처럼 지정.
- **초기 색인은 느립니다**(CPU 임베딩 수만 장 = 수 시간~수일). 그동안에도
  날짜/장소/OCR/얼굴 검색은 즉시 되고, 의미 검색은 색인된 만큼 점진 개선됩니다.
  폰에서 올리는 새 사진은 폰이 임베딩을 만들어 주면(업로드 페이지 옵션) 태블릿 부담이 없습니다.
- 안정 운용: 배터리 최적화 예외 + 안드로이드 12+는 phantom process killer 해제
  (`adb shell settings put global settings_enable_monitor_phantom_procs false`).

### 메모리 인식 색인 (모든 플랫폼)

임베딩 색인은 별도 백그라운드 스레드가 **여유 RAM을 보며 천천히** 진행합니다:
여유 2GB↑면 배치 16, 1~2GB면 8, 0.5~1GB면 4, 그 미만이면 일시정지 후 재시도.
저사양 기기는 배치 사이에 쉬어(듀티 사이클 `EMBED_DUTY`, 기본 0.5) 기기 반응성을
유지합니다. 진행상황은 상단 상태줄에 "🧠 AI 색인 n/m"으로 표시되고, DB에 즉시
저장되므로 껐다 켜도 이어서 진행됩니다.

## 아키텍처 메모

- **AI 하이브리드**: 색인은 로컬 CLIP(`clip-ViT-B-32`) — 사진이 기기를 떠나지 않음.
  한국어 질의는 영어로 변환(Claude API 또는 내장 사전) 후 CLIP 네이티브 인코더로 검색.
- **AI 선택형**: `AI_SEARCH=0` 또는 torch 미설치 시 메타데이터 검색으로 자동 폴백 — 라즈베리파이 대응.
- **얼굴 인식**: OpenCV 내장 YuNet(검출)+SFace(인식) — torch 불필요, 모델 ~37MB 자동 다운로드,
  라즈베리파이에서도 동작. `FACE_SEARCH=0`으로 끌 수 있음. 이름은 클러스터에 유지됨.
- **수만 장 대응**: SQLite + 월 단위 지연 로딩 + 썸네일 캐시.
- **휴지통**: 파일을 `data/trash/`로 이동 — 복원 가능, 영구삭제 전까지 안전.

## 폰 사진 전송 (2단계)

PC 화면의 **📱 폰 연결** 탭에서 모든 안내를 볼 수 있습니다.

1. **수동 업로드**: 폰 카메라로 QR을 찍으면 `http://<PC IP>:8765/upload` 업로드 페이지가 열립니다.
   수백 장을 한 번에 선택해도 되고, 원본 바이트를 그대로 저장하므로 화질·EXIF·GPS가 100% 보존됩니다.
   서버에 이미 있는 사진(해시 일치)은 자동으로 건너뜁니다. iPhone HEIC도 그대로 색인됩니다.
2. **자동 백업 (와이파이 진입 시)**: 브라우저는 백그라운드 업로드가 불가능하므로,
   내장 **WebDAV 서버**(`/webdav`)에 검증된 자동백업 앱을 연결합니다 —
   **PhotoSync**(iOS/Android) 또는 **FolderSync**(Android)에서 WebDAV 대상으로 주소를 넣고
   "지정 와이파이 연결 시 자동 전송"을 켜면, 폰이 집 와이파이에 들어올 때마다
   새 사진만 백그라운드로 올라갑니다. 업로드분은 `photos/MobileBackup/`에 저장·자동 색인됩니다.

## 구글 포토 백업 (Google Takeout 자동 처리)

> 구글은 계정 직접 로그인이나 Photos API로 **전체 라이브러리를 내려받는 것을 막아두었고**
> (2025년 4월 API 제한), API 다운로드는 **위치정보(GPS)를 제거**합니다.
> 메타데이터를 보존하는 유일한 공식 경로가 **Google Takeout**입니다.

1. <https://takeout.google.com> 에서 **Google 포토**만 선택해 내보내기(zip)
2. 받은 `takeout-*.zip`을 그대로 `photos/` 폴더에 넣기 (여러 조각 zip 모두)
3. 서버가 색인할 때 **자동으로 압축 해제** → 사이드카 JSON의 GPS·촬영시각을 살려 색인

- 이미 풀려 있는(같은 경로) 파일은 **건너뛰어** 중복 추출을 막습니다.
- 처리된 zip은 `photos/_zips_done/`으로 옮겨집니다(원하면 삭제). zip-slip 등 경로 공격을 차단합니다.
- 진행 상황은 화면 상단 색인 상태에 "Takeout 압축 해제 n/m"으로 표시됩니다.
